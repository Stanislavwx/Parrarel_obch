# Лабораторна робота №4

У програмі реалізовано все, що вимагається в методичці:

- послідовний LU-розклад;
- LU-розклад через `#pragma omp for`;
- LU-розклад через `#pragma omp task`;
- заповнення матриці випадковими числами у заданому діапазоні;
- перевірка правильності за формулою `A = L * U`;
- порівняння часу для `n = 10, 100, 500, 1000`.

## Як працює програма

1. Користувач вводить мінімальне і максимальне випадкове значення.
2. Програма створює матриці розмірів `10`, `100`, `500`, `1000`.
3. Для кожної матриці виконується LU-розклад трьома способами.
4. Після цього перевіряється правильність розкладу.
5. У кінці виводиться таблиця з часом роботи і похибкою.

## Як зібрати

```bash
g++ main.cpp -O2 -fopenmp -o lu_lab
```

або

```bash
make
```

## Як запустити

```bash
./lu_lab
```

Після запуску потрібно ввести діапазон випадкових чисел, наприклад:

```text
-10 10
```

## Коротко про код

У файлі [main.cpp](main.cpp) є тільки основні функції:

- `fillRandom()` — заповнення матриці випадковими числами;
- `luSequential()` — послідовний LU-розклад;
- `luParallelFor()` — LU-розклад через `omp for`;
- `luParallelTask()` — LU-розклад через `omp task`;
- `checkLU()` — перевірка правильності.

Кількість потоків зараз зафіксована в коді:

```cpp
const int THREADS = 4;
```

Якщо треба, це число можна легко змінити.

## Отримані результати

Для запуску з діапазоном `[-10; 10]` отримано такі результати:

| n | Повторів | Seq, мс | omp for, мс | omp task, мс |
|---|---:|---:|---:|---:|
| 10 | 2000 | 0.0051 | 0.0541 | 0.1469 |
| 100 | 200 | 0.4923 | 0.3296 | 10.7030 |
| 500 | 5 | 74.1474 | 21.1839 | 269.1671 |
| 1000 | 2 | 869.3199 | 184.6833 | 965.4379 |

Похибка перевірки для всіх трьох реалізацій дорівнювала `0.0000` після округлення.

## Висновок

Для малих матриць паралельні варіанти не дають виграшу через накладні витрати. Для більших матриць найкраще показав себе варіант через `omp for`. Варіант через `omp task` теж реалізований правильно, але в цій задачі працює повільніше.
