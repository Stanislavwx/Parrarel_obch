# Короткий звіт до лабораторної роботи №3

## Тема
Синхронізація в OpenMP-програмах.

## Мета роботи
Навчитися використовувати засоби синхронізації потоків в OpenMP та реалізувати паралельне обчислення числового ряду.

## Індивідуальне завдання
Варіант 1. Написати програму, яка наближено обчислює значення ряду

\[
\sum_{n=1}^{\infty}\frac{(-1)^{n+1}x^n}{n} = \ln(1 + x), \quad x \in (-1, 1]
\]

з використанням явної синхронізації потоків та технології замків.

## Опис програми
Було створено програму `lab3_var1.c`, яка:

- зчитує значення `x`;
- зчитує кількість членів ряду;
- зчитує кількість потоків;
- дозволяє вибрати метод синхронізації:
  - `atomic`;
  - `lock`.

У кожному потоці обчислюється окремий член ряду, після чого результат додається до спільної змінної суми.  
У першому випадку використовується директива `#pragma omp atomic`, у другому випадку - замок `omp_lock_t`.

## Компіляція

```bash
gcc -fopenmp lab3_var1.c -lm -o lab3_var1
```

## Приклад результату
Для `x = 0.5`, `N = 100000`, `4` потоки:

- наближене значення: `0.405465108108`;
- точне значення: `0.405465108108`;
- абсолютна похибка: `0.000000000000`.

Для `x = 1`, `N = 1000000`, `4` потоки:

- наближене значення: `0.693146680560`;
- точне значення: `0.693147180560`;
- абсолютна похибка: `0.000000500000`.

## Висновок
У ході роботи було реалізовано паралельне обчислення ряду для функції `ln(1 + x)` засобами OpenMP.  
Програма коректно працює з двома способами синхронізації: через `atomic` і через `lock`.  
Отримані результати збігаються з теоретичними значеннями з малою похибкою.
